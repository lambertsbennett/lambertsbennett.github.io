Concurrent Biological Sequence Processing with Go

Many sequence manipulations are "embarrassingly parallelizeable" and Go makes concurrency incredibly simple. As part of our machine learning efforts in the Armbrust Lab I have been developing a number of preprocessing tools for DNA/RNA sequence manipulation.

GoMrLBP

Multi-resolution local binary patterns for biological sequence data in Go. Inspired by Kouchaki et al. (2019), I implemented multi-resolution local binary patterns in Go. Beyond the original C++ implementation, the Go implementation can run on multiple cores and enables large volumes of sequence to be processed in seconds to minutes.

GoKmer

Concurrent k-mer expansion of biological sequence data in Go. When pairing NLP techniques with biological sequence data, a frequent operation is to explode a sequence into its constitutive k-mers. This can be a very slow and memory intensive operation if you're not careful. Here I implemented a very simple tool in Go that concurrently expands sequences in fasta format into their k-mers. Large assemblies can be processed on the order of seconds to minutes on an 8 core laptop.

GoKmer

Concurrent one-hot encoding of biological sequences in Go. Another common operation that preceeds application of deep learning techniques on biological sequences is one-hot encoding input sequences. Here I've put together a very simple one-hot encoding helper tool.

Image Processing in MATLAB

  1. Pond fractals in a tidal flat (link). In this work, I processed images captured by a cell phone that was suspended above a tidal flat. This was a challenging processing task, as the changes in pixel intensity that many algorithms rely on for object detection were not very pronounced in this system.
